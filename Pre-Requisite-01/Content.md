SQL Day 1: Introduction to Data
What is Data?

Data is a collection of facts, figures, observations, or information that can be stored, processed, and analyzed to gain meaningful insights.

Data can exist in different forms such as:

Text (Names, Addresses)
Numbers (Marks, Salary, Age)
Images (Photos, X-rays)
Audio (Voice recordings, Songs)
Videos (Movies, CCTV footage)
Documents (PDFs, Word files)
Definition

Anything that exists around us and can be recorded, stored, and processed is called Data.

Importance of Data

In today's digital world, almost everything runs on data.

Famous Quote

"Data is the new oil."

This means that data has become one of the most valuable resources in the modern world, just like oil was during the industrial revolution.

Why is Data Important?
1. Better Decision Making

Companies analyze data to make business decisions.

Example:
Amazon recommends products based on your previous purchases.

2. Understanding Customers

Businesses use customer data to understand:

Customer preferences
Buying patterns
Customer satisfaction

Example:
Netflix recommends movies based on your watch history.

3. Improving Efficiency

Organizations use data to improve their operations and reduce costs.

Example:
Delivery companies use traffic data to find the shortest route.

4. Predicting Future Trends

Data helps organizations predict future events.

Example:
Weather forecasting uses historical weather data.

5. Automation and Artificial Intelligence

AI systems work entirely on data.

Example:
ChatGPT, Self-driving cars, Recommendation systems.

Industry	Data Used

Education	Student marks

Types of Data

Data is mainly classified into three categories:

Structured Data
Semi-Structured Data
Unstructured Data
1. Structured Data

Structured data is data that is organized in a predefined format, usually in rows and columns.

It follows a fixed schema.

Example
ID	Name	Age	City
1	Praveen	21	Chittoor
2	Uday	20	Tirupati
3	Nithin	22	Bangalore
Characteristics
Organized format
Easy to search
Easy to store
Easy to analyze
Stored in databases
Examples
Employee records
Student information
Bank transactions
Product details
Databases Used
MySQL
Oracle
PostgreSQL
SQL Server
2. Semi-Structured Data

Semi-structured data does not follow a strict table structure but contains tags or keys to organize the information.

Examples
{
  "name": "Praveen",
  "age": 21,
  "city": "Chittoor"
}
Characteristics
Flexible structure
Self-describing
Easy to exchange between applications
Examples
JSON files
XML files
Emails
Log files
Databases Used
MongoDB
Firebase
Cassandra
3. Unstructured Data

Unstructured data has no predefined format or organization.

Examples
Images
Videos
Audio files
Social media posts
PDF documents
Characteristics
Very large in volume
Difficult to process
Requires advanced technologies like AI and Big Data
Examples
YouTube videos
CCTV footage
WhatsApp messages
Instagram photos
Comparison of Data Types
Feature	Structured	Semi-Structured	Unstructured
Format	Fixed	Flexible	No Format
Schema	Yes	Partial	No
Storage	SQL Databases	NoSQL Databases	Data Lakes
Analysis	Easy	Moderate	Difficult
Example	Tables	JSON	Images
Why Should We Learn About Data Before SQL?

SQL stands for Structured Query Language.

SQL is used to:

Store structured data.
Retrieve data.
Update data.
Delete data.
Analyze data.

Without understanding what data is and how it is organized, learning SQL becomes difficult.

Key Takeaways

✅ Data is information that can be stored and processed.

✅ Data is the backbone of modern technology.

✅ Data is classified into:

Structured Data
Semi-Structured Data
Unstructured Data

✅ SQL mainly works with Structured Data stored in relational databases.
